# Design + Tasks 拆解 Prompt

## 使用时机

当 proposal 已确定，需要详细拆解任务时使用此 prompt。

## Prompt 模板

```
在不改变 proposal 范围的前提下，把 tasks.md 拆到 0.5–1 天/任务。

## 每条任务必须包含

1. **目的**：这条任务要达成什么
2. **改动文件/模块**：具体涉及哪些文件或模块
3. **关键实现点**：实现的核心逻辑
4. **对应的验收/测试**：如何验证任务完成

## 额外要求

- 标注任务之间的依赖关系
- 标注错误处理方式
- 确保每条任务可独立验收
- 不能完成就重拆
```

## 输出要求

更新 `changes/[change-id]/tasks.md` 文件。

## 审计日志要求

**步骤开始时**：在 AUDIT.md 时间线中添加

```markdown
| 设计拆解 | [开始时间] | | | 进行中 |
```

**步骤结束时**：
1. 更新时间线（记录结束时间和耗时）
2. 在 `.audit/` 创建快照 `002_tasks_breakdown.md`
3. 记录任务拆分的决策说明（为什么这样拆分）
